Data Types

Efficiently Representing Values and Tagging

在 javascript 的规范中，所有的数都是用 8 字节 double 浮点型表示的，在 V8 中为了提高效率，只用了 32 bit，如下最低为用来区分 Small Integers 和 Objects

在 64 bit 情况下，使用 32 bit 表示整数

长度大于 31 bits 的整数 boxed(想不出来一个准确的翻译)

Object Properties

  • Object 是属性的集合
  • 属性名字必须为字符串
  • Javascript 可以看作具有长度属性的 Object

HashTables and Hash Codes

Map, Set, WeakSet, 和 WeakMap 都是通过 HashTable 实现的，使用 hash 函数将给定的 key 映射到 HashTable 中。在 V8 中，hash 函数会返回一个随机值，用于存储。如果 key 是 object ，那返回的对象将存储在 object 的 private symbol 上。

在 Object 添加到 hashtable 前不需要为其保留内存，V8 也可以像使用 Inline Caches 去寻找 Object 属性信息一样去寻找 hashcode，这在 monomorphic IC 查找中非常有效，即 key 拥有相同的 hidden class，但是真实的程序中 key 往往具有不同的 hidden class，导致 megamorphic IC 查找效率降低。

Hiding the hash code

elements backing store 中的数组大小没有上限，而 properties backing store 数组大小最大为 1022，当超过这个限制的时候 properties backing store 会使用字典存储。

The properties backing store is empty

直接把 hashcode 存储在 JSObject 的属性指向的内存。

The properties backing store is an array

使用 Array 的 length 字段的 11-31 位存储 hashcode

The properties backing store is a dictionary

将字典大小增加一个 word 并将 hashcode 存储在 header 后。

Fast, In-Object Properties

V8 使用 maps 来描述对象的数据结构，map 包含对象的大小和构造函数及原型信息。使用相同构造函数创建的对象具有相同的属性集，具有相同结构的对象共享 map。通过 transition descriptor 来添加其他属性，transition descriptor指向其他 map。

Assigning Properties inside Constructor Call

  • Point starts out without any fields with M0
  • this.x =x -> map pointer set to M1 and value x is stored at offset 12 and "x" Transition descriptor added to M0
  • this.y =y -> map pointer set to M2 and value y is stored at offset 16 and "y" Transition descriptor added to M1

Assigning More Properties Later

  • assigning z later
    • create M3, a copy of M2
    • add Transition descriptor to M2
    • add Field descriptor to M3

Assigning Same Properties in Different Order

  • M0 将会有两个 transitions 链， 形成 transitions 树
  • 如果使用同一个构造函数创建含有大量的且顺序不固定属性的对象或者删除属性时， V8 将会进入字典模式，将属性存储在 hashtable 中，以防止创建大量 map。

In-object Slack Tracking

当一个 constructor 创建 object 时，虽然不知道其大小，但会保留 32 个 fast properties 大小，当使用同一个 constructor 创建超过 8 个 object 时， v8 会从最初的 map 遍历 transition 树，来获取当前 object 的最大 size，后面新创建相同对象的将会根据取得的大小来分配内存，已经初始化的 object 会被 garbage collector 认为是 free space，因为 maps 指定了其大小，所以其不会被回收，当 In-object Slack Tracking结束的时候，新的实例的大小被写入 transition 树的 maps 里，所以一些 object 将会自然变小，当为 object 创建新的属性时，会通过分配一个 overflow array 来存放，当新的属性创建时 overflow array 会 reallocated

Methods And Prototypes

Assigning Functions to Properties

  • properties pointing to Functions are handled via constant functions descriptor
  • constant_function descriptor indicates that value of property is stored with descriptor itself rather than in the object
  • pointers to functions are directly embedded into optimized code
  • if distance is reassigned, a new map has to be created since the Transition breaks

Assigning Functions to Prototypes

  • V8 represents prototype methods (aka class methods) using constant_function descriptors
  • calling prototype methods maybe a tiny bit slower due to overhead of the following:
    • check receiver’s map (as with own properties)
    • check maps of prototype chain (extra step)
  • the above lookup overhead won’t make measurable performance difference and shouldn’t impact how you write code

    Numbered Properties

    numbered properties 被处理和排序的方法和其他情况不同，v8 的 Array 可以具有 holes，当访问时会返回 undefined，因为 properties 和 elements 是被分开存储的，所以即使对象陷入了字典模式，访问速度仍很快

    Arrays

    V8 有两种方法存储数组，fast elements 和 dictionary elements

    Fast Elements

  • 紧凑型键组
  • 线性存储区
  • 连续
  • 0 based
  • 小于 64k

Dictionary Elements

  • hash 表存储
  • 访问慢
  • 适合存放元素稀疏的数组
  • 存放容量大

Named property types

  • In-object properties 直接存储在 object 里，便于访问
  • Fast properties 存放在 properties store 里，所有的信息被存储在 Hiddenclass 上的 descriptor array 里
  • Slow properties 存储在 object 包含的 properties 字典里，信息不再与 hiddenclass 共享，在增加大量属性 或者删除属性时的效率
